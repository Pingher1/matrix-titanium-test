## DEEP SCAN 2: SYSTEM ARCHITECTURE & PROGRESS LOGS

This serves as a persistent snapshot of the architectural decisions, components, and backend microservices that have been constructed for the Kronos Avatar Forge project. 

### 1. Global Application Architecture
The application has been restructured from a single-page viewer into a scalable, multi-suite React application using React Router and dynamic JSON configurations.
*   **`public/app-menus/menu-v1.json`**: The global routing configuration mapping the three major application suites (`Avatar Studio`, `Storybook Library`, and `Barbie World`).
*   **`src/App.tsx`**: The root entrypoint wrapped in `BrowserRouter` and `MenuLoader`. Mounts the 3D Canvas, the Storybook players, and the Barbie placeholder widgets depending on the route.

### 2. 3D React Three Fiber Engine (Avatar Studio)
The core 3D generation and rendering engine built on top of Three.js and `@react-three/fiber`.
*   **`src/components/ModelLoader.tsx`**: The hero loader for `.glb` files. It handles skin pigmentation materials, toggling mesh visibility for clothing, applying shadows, and bridging the animations from the GLB or the procedural registry. 
*   **`src/lib/animationRegistry.ts`**: A procedural animation fallback system calculating mathematical Three.js `AnimationClip` KeyFrame tracks when a rig doesn't have the baked-in animation clip.

### 3. Storybook TTS & Karaoke Pipeline
The interactive storybook system capable of processing AWS Polly audio and timestamp arrays for karaoke-style highlights and physical Avatar bridging.
*   **`src/lib/eventBus.ts`**: An ultra-fast event emitter using `mitt` for asynchronous global events.
*   **`src/lib/AvatarActionBridge.ts`**: Converts `eventBus` semantic actions into `ModelLoader.tsx` window bindings to make the Avatar move.
*   **`src/hooks/useSpeechSync.ts`**: The engine for the Karaoke logic, evaluating audio buffers to manipulate jaw frames.

### 4. Backend Express Microservices
The backend API (`server/`) handling heavy AWS S3 data-shuttling, validation, and AWS Polly generation.
*   **`server/src/index.ts`**, **`server/src/tts/pollyExpress.ts`**, **`server/src/upload/presign.ts`** handling local and AWS routing.
